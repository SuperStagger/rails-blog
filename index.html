📋 Покроковий гайд

Як створити Rails блог

Кожен крок — за офіційним Getting Started з Rails.

01
Встановлення Ruby та Rails
Перевірте та встановіть Ruby (≥ 3.2), RubyGems і Rails через gem install. Рекомендується використовувати rbenv або RVM для управління версіями.
# Перевірити версію Ruby
ruby --version

# Встановити Rails
gem install rails

# Перевірити версію Rails
rails --version
# => Rails 8.x.x
Примітка: Для Windows рекомендується WSL2 (Ubuntu). Для macOS — Homebrew + rbenv.
02
Створення нового Rails проекту
Команда rails new генерує повну структуру програми з усіма необхідними файлами та залежностями.
# Створити новий Rails додаток
rails new blog

# Перейти в папку проекту
cd blog

# Запустити сервер
rails server
blog/ ├── app/ │ ├── controllers/ │ ├── models/ │ └── views/ ├── config/ │ └── routes.rb ├── db/ ├── Gemfile └── README.md
03
Генерація Article ресурсу
Scaffold автоматично створює модель, контролер, в'юхи та маршрути для CRUD операцій над статтями блогу.
# Scaffold для Article (заголовок + текст)
rails generate scaffold Article \
  title:string body:text

# Застосувати міграцію бази даних
rails db:migrate
04
Налаштування маршрутів
Відредагуйте config/routes.rb, щоб встановити кореневий маршрут на список статей.
# config/routes.rb
Rails.application.routes.draw do
  root "articles#index"
  resources :articles
end
Перевірте маршрути: Запустіть rails routes щоб побачити всі доступні URL-шляхи.
05
Додавання коментарів
Розширте блог функцією коментарів — це демонструє зв'язки між моделями (has_many / belongs_to) у Rails.
# Генерація моделі Comment
rails generate model Comment \
  commenter:string body:text \
  article:references

rails db:migrate
# app/models/article.rb
class Article < ApplicationRecord
  has_many :comments, dependent: :destroy
  validates :title, presence: true
  validates :body, presence: true,
                   length: { minimum: 10 }
end
06
Базова безпека (HTTP Auth)
Захистіть адміністративні дії (create, edit, destroy) простою HTTP Basic автентифікацією, що вбудована у Rails.
# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  http_basic_authenticate_with \
    name: "admin",
    password: "secret",
    except: [:index, :show]

  # ... решта дій
end
Готово! Відкрийте http://localhost:3000 і ваш блог працює ✓

01
Генерація моделі Comment
Створюємо модель Comment з посиланням на Article через article:references — Rails автоматично додасть зовнішній ключ і індекс.
# Генерація моделі Comment
rails generate model Comment \
  commenter:string \
  body:text \
  article:references

# Застосувати міграцію
rails db:migrate
02
Зв'язки між моделями (has_many / belongs_to)
Один пост має багато коментарів, кожен коментар належить конкретному посту — це класичний зв'язок one-to-many у Rails.
# app/models/article.rb
class Article < ApplicationRecord
  has_many :comments, dependent: :destroy
  validates :title, presence: true
  validates :body, presence: true,
                   length: { minimum: 10 }
end

# app/models/comment.rb
class Comment < ApplicationRecord
  belongs_to :article
  validates :body, presence: true
end
Важливо: validates :body, presence: true — обов'язкова валідація за умовами завдання. Коментар без тексту не збережеться.
03
Маршрути — вкладені ресурси
Коментарі є вкладеними ресурсами всередині статей. Це відображає їх залежність у URL-структурі.
# config/routes.rb
Rails.application.routes.draw do
  root "articles#index"
  resources :articles do
    resources :comments
  end
end
URL коментарів матиме вигляд: /articles/1/comments
04
CommentsController
Контролер обробляє створення та видалення коментарів. Знаходить батьківську статтю через params[:article_id].
rails generate controller Comments
# app/controllers/comments_controller.rb
class CommentsController < ApplicationController
  def create
    @article = Article.find(params[:article_id])
    @comment = @article.comments.create(comment_params)
    redirect_to article_path(@article)
  end

  def destroy
    @article = Article.find(params[:article_id])
    @comment = @article.comments.find(params[:id])
    @comment.destroy
    redirect_to article_path(@article)
  end

  private
    def comment_params
      params.require(:comment).permit(:commenter, :body)
    end
end
05
View — форма та список коментарів
Додайте форму та список коментарів у в'юху статті app/views/articles/show.html.erb.
<!-- Список коментарів -->
<h2>Коментарі</h2>
<% @article.comments.each do |comment| %>
  <div>
    <strong><%= comment.commenter %></strong>
    <p><%= comment.body %></p>
    <%= link_to "Видалити",
      article_comment_path(@article, comment),
      data: { turbo_method: :delete } %>
  </div>
<% end %>

<!-- Форма нового коментаря -->
<h2>Новий коментар</h2>
<%= form_with model: [@article, Comment.new] do |f| %>
  <p>
    <%= f.label :commenter %><br>
    <%= f.text_field :commenter %>
  </p>
  <p>
    <%= f.label :body %><br>
    <%= f.text_area :body %>
  </p>
  <%= f.submit %>
<% end %>
Готово! Запустіть rails server і перейдіть на будь-яку статтю — форма коментарів вже працює ✓

01
Встановлення Devise
Додайте gem до Gemfile та встановіть його через bundler.
# Gemfile
gem "devise"
bundle install
rails generate devise:install
Після інсталяції термінал покаже інструкції — додайте root_url та flash-повідомлення до application.html.erb.
02
Генерація моделі User
Devise генерує модель User з усіма необхідними полями для автентифікації та міграцію бази даних.
rails generate devise User
rails db:migrate
# app/models/user.rb — згенерований Devise
class User < ApplicationRecord
  devise :database_authenticatable, :registerable,
         :recoverable, :rememberable, :validatable
end
03
Захист контролерів
Додайте before_action щоб лише авторизовані користувачі могли створювати та редагувати статті.
# app/controllers/articles_controller.rb
class ArticlesController < ApplicationController
  before_action :authenticate_user!,
    except: [:index, :show]

  # ... решта дій
end
04
Навігація — вхід / вихід

⚙️ ActiveAdmin — Адмін-панель

Встановлення адміністративної панелі

ActiveAdmin — готова адмін-панель для управління даними блогу без написання додаткового коду.

01
Встановлення ActiveAdmin
Додайте необхідні gems до Gemfile. ActiveAdmin вимагає Devise для автентифікації адміністраторів.
# Gemfile
gem "activeadmin"
gem "ransack"   # для пошуку
gem "kaminari"  # для пагінації
bundle install
rails generate active_admin:install
rails db:migrate
rails db:seed
Після seed: адмін акаунт — admin@example.com / пароль password. Змініть у db/seeds.rb!
02
Реєстрація моделей в адмінці
Додайте Article і Comment до панелі адміністратора щоб керувати ними через UI.
rails generate active_admin:resource Article
rails generate active_admin:resource Comment
# app/admin/articles.rb
ActiveAdmin.register Article do
  permit_params :title, :body

  index do
    selectable_column
    id_column
    column :title
    column :created_at
    actions
  end
end
03
Відкрити адмін-панель
Запустіть сервер і перейдіть за адресою — панель вже готова до роботи.
rails server
Адмін-панель доступна за: http://localhost:3000/admin — увійдіть з кредами з seeds.rb
04
Залити зміни на GitHub
Зафіксуйте всі зміни та відправте в існуючий репозиторій.
git add .
git commit -m "Add Devise auth and ActiveAdmin panel"
git push origin main
